# MyGo 能效标签与缺陷检测系统 — 项目详细方案

## 一、项目概述

### 1.1 项目名称
MyGo 能效标签与缺陷检测系统

### 1.2 项目定位
基于 OpenHarmony 平台的工业级智能质检系统，利用深度学习技术实现产品能效标签的自动化检测，覆盖标签识别、缺陷检测、位置校验和参数提取等全流程质检需求。

### 1.3 项目周期
- 需求分析与设计：2 周
- 核心功能开发：4 周
- AI 模型训练与优化：3 周
- 系统集成与测试：2 周
- 部署与验收：1 周

## 二、需求分析

### 2.1 业务需求

#### 2.1.1 业务痛点
| 编号 | 痛点 | 影响 |
|------|------|------|
| BP-01 | 人工检测效率低下 | 产线产能受限，人均检测速度约 30 秒/件 |
| BP-02 | 质检标准不统一 | 不同检测员判断标准存在差异，漏检率波动大 |
| BP-03 | 质检数据难追溯 | 纸质记录易丢失，无法支撑质量趋势分析 |
| BP-04 | 人力成本高 | 需要大量专职质检人员，培训和管理成本高 |
| BP-05 | 合规风险 | 能效标签不合规可能导致产品下架和罚款 |

#### 2.1.2 业务目标
1. 实现能效标签自动化检测，检测速度提升 5-10 倍
2. 统一质检标准，消除人为判断差异
3. 全流程数据记录，支持质量追溯和分析
4. 降低 60%-80% 的质检人力成本
5. 确保产品能效标签合规率达到 99% 以上

### 2.2 功能需求

#### FR-01 手动检测模式
- **FR-01-01** 图片选择：支持从设备相册选择待检测图片
- **FR-01-02** 相机拍摄：集成设备相机实时拍摄能效标签
- **FR-01-03** 测试图片：提供 13 张预设测试图片用于验证
- **FR-01-04** 实时分析：上传图片后实时调用 AI 引擎进行分析
- **FR-01-05** 结果展示：展示等级、缺陷、位置、OCR 等综合结果

#### FR-02 工业产线模式
- **FR-02-01** 自动拍照：按照设定间隔（5/10/30/60秒）自动拍照
- **FR-02-02** 自动检测：拍照后自动触发 AI 检测流程
- **FR-02-03** 实时统计：显示检测总数、合格率、缺陷分布
- **FR-02-04** 会话管理：追踪检测会话时长和状态
- **FR-02-05** 最近结果：实时显示最近若干条检测结果详情

#### FR-03 能效等级识别
- **FR-03-01** 颜色分析：基于 HSV 颜色空间分析标签等级色条
- **FR-03-02** 等级判定：自动判定 1-5 级能效等级
- **FR-03-03** 置信度输出：输出等级判定的置信度分数
- **FR-03-04** 多策略回退：主策略失败时自动尝试备选 ROI 和放大策略

#### FR-04 缺陷检测
- **FR-04-01** 破损检测：检测标签表面的破损缺陷（DAM）
- **FR-04-02** 污渍检测：检测标签表面的污渍污染（STA）
- **FR-04-03** 褶皱检测：检测标签的褶皱变形（WRI）
- **FR-04-04** 多缺陷支持：支持单张图片同时检测多种缺陷

#### FR-05 位置校验
- **FR-05-01** 中心定位：计算标签在画面中的中心坐标
- **FR-05-02** 偏差计算：计算标签中心与画面中心的偏移百分比
- **FR-05-03** 合规判定：偏差 ≤10% 判定为位置合规

#### FR-06 OCR 参数提取
- **FR-06-01** 文字识别：识别标签上的全部文字内容
- **FR-06-02** 能效参数：提取能效参数值（X.X 格式数字）
- **FR-06-03** 待机功率：提取待机功率值
- **FR-06-04** 等级辅助：通过 OCR 识别的数字辅助等级判定

#### FR-07 数据管理
- **FR-07-01** 自动记录：每次检测完成自动生成检测记录
- **FR-07-02** 历史查询：支持按时间范围浏览历史检测记录
- **FR-07-03** 多维筛选：按产品型号、检测状态、日期筛选
- **FR-07-04** 统计分析：合格率统计、缺陷分布分析、趋势图表
- **FR-07-05** CSV 导出：支持将检测数据导出为 CSV 格式文件

#### FR-08 系统管理
- **FR-08-01** 检测参数配置：精度（50%-95%）、速度（30%-90%）参数设置
- **FR-08-02** 服务器配置：后端服务 URL 地址配置
- **FR-08-03** 产线参数：拍照间隔、自动化参数调整
- **FR-08-04** 应用信息：版本号、设备状态等系统信息展示

### 3.4 AI 检测引擎设计

#### 3.4.1 检测流程
```
输入图片
    │
    ▼
YOLOv8 目标检测
    │
    ├── 检测到标签 (label/nor)
    │       │
    │       ├── 检测到缺陷 (break/stain/wrinkle)
    │       │       │
    │       │       ▼
    │       │   返回缺陷结果（不合格）
    │       │
    │       └── 无缺陷
    │               │
    │               ├── HSV 颜色等级检测
    │               │       │
    │               │       ▼
    │               │   等级 1-5 + 置信度
    │               │
    │               ├── PaddleOCR 文字识别
    │               │       │
    │               │       ▼
    │               │   能效参数 + 待机功率
    │               │
    │               └── 综合判定
    │                       │
    │                       ▼
    │               合格/不合格 + 完整报告
    │
    └── 未检测到标签
            │
            ▼
        返回未检测到标签（不合格）
```

#### 3.4.2 HSV 颜色等级检测算法
1. 裁剪标签区域，提取 ROI
2. 转换到 HSV 颜色空间
3. 自适应饱和度阈值（40→25→15 递减尝试）
4. 定位等级色条箭头位置
5. 采样箭头区域高饱和度像素的色调值
6. 色调直方图统计，映射到 1-5 级
7. 多策略回退：标准 ROI → 宽 ROI → 窄 ROI → 4 倍放大 → 全区域搜索

色调-等级映射表：
| 色调范围 (H) | 等级 |
|-------------|------|
| 168-12 (红) | 5 级 |
| 12-24 (橙) | 4 级 |
| 24-36 (黄橙) | 3 级 |
| 36-65 (黄绿) | 2 级 |
| 65-105 (绿) | 1 级 |

## 五、数据设计

### 5.1 数据模型

#### 检测结果（DetectionResult）
```typescript
interface DetectionResult {
  success: boolean;           // 检测是否成功
  grade: number | null;       // 能效等级 1-5
  gradeMethod: string | null; // 等级检测方法 (color/ocr)
  gradeConfidence: number;    // 等级置信度
  energyParam: number | null; // 能效参数
  standbyPower: number | null;// 待机功率
  defects: {
    isDamaged: boolean;       // 是否破损
    isStained: boolean;       // 是否污渍
    isWrinkled: boolean;      // 是否褶皱
  };
  position: {
    isCorrect: boolean;       // 位置是否合规
    x: number;                // 标签中心 X
    y: number;                // 标签中心 Y
    deviation: number;        // 偏差百分比
  };
  isPass: boolean;            // 是否合格
  detections: Detection[];    // 原始检测结果数组
  labelFound: boolean;        // 是否检测到标签
  hasDefect: boolean;         // 是否有缺陷
  labelConfidence: number;    // 标签检测置信度
  labelBbox: number[];        // 标签边界框
}
```

#### 检测目标（Detection）
```typescript
interface Detection {
  class: string;    // 目标类别
  confidence: number; // 置信度
  bbox: number[];    // 边界框 [x1,y1,x2,y2]
}
```

#### 历史记录（HistoryRecord）
```typescript
interface HistoryRecord {
  id: string;               // 记录 ID
  timestamp: string;        // 检测时间
  result: DetectionResult;  // 检测结果
  productName: string;      // 产品名称
  modelNumber: string;      // 产品型号
}
```

### 5.2 数据存储
- 当前版本：内存存储（Node.js 数组）
- 后续规划：SQLite（本地持久化）/ PostgreSQL（服务器端）

| 风险 | 等级 | 应对措施 |
|------|------|----------|
| 模型精度不足 | 中 | 持续收集数据，优化训练 |
