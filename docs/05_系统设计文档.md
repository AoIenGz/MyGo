## 一、系统架构设计

### 1.2 架构设计原则

| 原则 | 说明 |
|------|------|
| 分层解耦 | 前端、后端、AI 引擎三层独立部署，通过标准接口通信 |
| 技术异构 | 前端 ArkTS + 后端 Node.js + AI 引擎 Python，各取所长 |

**图片分析流程**：
1. 解析 Base64 图片数据
2. 保存为临时文件（`uploads/upload_{timestamp}.jpg`）

4. 捕获 stdout 输出，解析最后一行 JSON
5. 更新缓存结果
6. 自动存入历史记录
7. 清理临时文件
8. 60 秒超时保护

### 2.3 AI 引擎模块设计

#### 2.3.1 检测脚本（detect_api.py）

**模块结构**：
```python
detect_api.py
├── load_model()           # YOLOv8 模型加载
├── detect_grade(crop)     # HSV 颜色等级检测
│   ├── _detect_with_roi() # ROI 区域检测
│   └── _analyze_roi()     # HSV 色调分析
├── extract_ocr(crop)      # PaddleOCR 文字提取
└── analyze_image(path)    # 主分析流程
```

#### 2.3.2 检测流程详细设计

```
输入: image_path
    │
    ▼
[1] 读取图片 (cv2.imread)
    │
    ▼
[2] YOLO 目标检测 (model(image_path))
    │
    ├── 遍历检测结果
    │   ├── 识别标签 (label/nor) → 记录最佳标签区域
    │   ├── 识别破损 (break)     → defects.isDamaged = True
    │   ├── 识别污渍 (stain)     → defects.isStained = True
    │   └── 识别褶皱 (wrinkle)   → defects.isWrinkled = True
    │
    ├── [3A] 未检测到标签 → 返回"未检测到标签"
    │
    ├── [3B] 检测到缺陷 → 返回缺陷结果（不合格）
    │
    └── [3C] 无缺陷 → 进入等级检测
         │
         ▼
    [4] HSV 颜色等级检测 (detect_grade)
         │
         ├── 标准 ROI 策略 (8%-58% H, 58% W)
         │   ├── 饱和度阈值 40
         │   ├── 饱和度阈值 25
         │   └── 饱和度阈值 15
         │
         ├── 宽 ROI 回退 (5%-65% H, 60% W)
         │
         ├── 窄 ROI 回退 (10%-55% H, 55% W)
         │
         ├── 4 倍放大回退
         │
         └── 全区域搜索 (5%-75% H, 70% W)
         │
         ▼
    [5] PaddleOCR 文字识别 (extract_ocr)
         │
         ├── 图片放大 4 倍
         ├── 文字识别
         ├── 提取能效参数 (X.X 格式)
         ├── 提取待机功率
         └── OCR 等级辅助 (置信度 > 0.8)
         │
         ▼
    [6] 综合判定
         │
         ├── 等级优先取颜色检测结果
         ├── 颜色检测失败则取 OCR 结果
         └── 合格判定: 有等级 + 位置合规
         │
         ▼
    输出: JSON 检测结果
```

#### 2.3.3 YOLO 检测模型设计

**模型信息**：
| 属性 | 值 |
|------|------|
| 架构 | YOLOv8n (nano) |
| 模型文件 | best.pt |
| 模型大小 | 6 MB |
| 输入尺寸 | 640 × 640 |
| 检测类别 | label, nor, break, stain, wrinkle |
| 训练框架 | Ultralytics |
| 推理框架 | PyTorch |

**类别定义**：
| 类别名 | 标识 | 说明 |
|--------|------|------|
| label | 正常标签 | 能效标签正常粘贴 |
| nor | 正常标签（备选） | 能效标签正常（训练标注差异） |
| break | 破损 | 标签表面破损/撕裂 |
| stain | 污渍 | 标签表面有污渍/污染 |
| wrinkle | 褶皱 | 标签表面有褶皱/起皱 |

#### 2.3.4 HSV 颜色等级检测算法

**算法核心**：通过分析能效标签等级色条的颜色色调来判定等级。

**等级-色调映射**：
```
色调 H (OpenCV 0-180)
    │
    ├── 0-12 / 168-180  →  5 级 (红色)
    ├── 12-24           →  4 级 (橙色)
    ├── 24-36           →  3 级 (黄橙色)
    ├── 36-65           →  2 级 (黄绿色)
    └── 65-105          →  1 级 (绿色)
```

**处理步骤**：
1. 从 YOLO 检测结果中裁剪标签区域
2. 提取 ROI 区域（排除边缘和标题栏）
3. 转换到 HSV 颜色空间
4. 按行扫描饱和度高于阈值的像素，定位等级色条
5. 跳过绿色标题栏区域
6. 定位等级箭头（色条右边缘延伸区域）
7. 采样箭头行的高饱和度像素色调
8. 统计色调直方图，多数投票确定等级

**自适应策略**：
- 饱和度阈值递减：40 → 25 → 15
- ROI 区域多策略回退（5 种策略）

## 六、性能设计

### 6.1 前端性能
- 列表分页加载，避免大数据量渲染
- 图片压缩后再上传
- 检测结果本地缓存

### 6.2 后端性能
- Express 中间件 JSON 解析限制 50MB
- Python 子进程池管理（规划）
- 检测结果内存缓存

### 6.3 AI 引擎性能
- YOLOv8n 轻量模型，推理时间 < 2 秒
- OCR 前 4 倍放大，平衡精度与性能
- HSV 分析纯 NumPy 运算，毫秒级完成
